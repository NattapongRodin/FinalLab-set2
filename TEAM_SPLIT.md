## การแบ่งงานหลัก

### **สมาชิกคนที่ 1: ไตรภพ ก๋องใจ**
รับผิดชอบงานหลักดังต่อไปนี้:
* **System Testing:** 
ทดสอบระบบการทำงานผ่าน docker ก่อน deploy
* **Railway Deployment:** 
รับผิดชอบการนำ Dockerized Services ทั้งหมดขึ้นไปรันบนแพลตฟอร์ม **Railway**
**Environment Management:**
ตั้งค่าตัวแปรสภาพแวดล้อม และดูแลการเชื่อมต่อระหว่าง Database และ Backend Service

### **สมาชิกคนที่ 2: นายนัฐพงศ์  รอดอินทร์**
รับผิดชอบงานหลักดังต่อไปนี้:
* **Frontend:**  
พัฒนาและปรับปรุงหน้าจอ Dashboard, Login และ Register ให้รองรับฟีเจอร์ใหม่ใน Set 2
* **Backend:** 
รับผิดชอบการเขียน Logic ในส่วนของ **Task Service** และ **User Service** เพื่อจัดการข้อมูลที่ซับซ้อนขึ้นและ ออกแบบและจัดการโครงสร้างตารางข้อมูลใน `auth-db` `task-db` และ `user-db`

## เหตุผลในการแบ่งงาน
แบ่งงานตาความถนัดส่วนบุคคล โดยแยกส่วนโครงสร้างพื้นฐาน (Infrastructure/Backend) ออกจากส่วนการนำเสนอข้อมูล (Frontend) เพื่อให้สามารถพัฒนาไปพร้อมกันได้ ทดสอบข้อมูลเหล่านั้นเพื่อนำไป deploy บน railway

## สรุปการเชื่อมโยงงานของสมาชิก
งานของสมาชิกทั้งสองคนเชื่อมต่อกันผ่าน **API Gateway ** โดยเมื่อทำในส่วนของ backend frontend เสร็จแล้ว ทดสอบระบบผ่าน postman และนำไป deploy บน railway
