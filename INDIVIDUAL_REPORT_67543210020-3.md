## สิ่งที่ได้ดำเนินการด้วยตนเอง
พัฒนาและปรับปรุงหน้า Dashboard, Login และ Register ให้รองรับฟีเจอร์ใหม่ใน Set 2 และเชื่อมต่อการทำงานกับ backend ได้ถูกต้อง
เขียน Logic ของ Task Service และ User Service เพื่อรองรับการจัดการข้อมูลที่ซับซ้อนขึ้น เช่น การสร้าง แก้ไข ลบ และดึงข้อมูลตามเงื่อนไขของระบบ

## ปัญหาที่พบและวิธีการแก้ไข
- โค้ดเดิมจาก Set 1 ใช้กับ Set 2 ไม่ได้ทั้งหมด ปรับ logic ใหม่ให้แต่ละ service ใช้เฉพาะฐานข้อมูลของตนเอง และใช้ user_id จาก JWT เป็นตัวอ้างอิงแทนการ query ข้ามฐานข้อมูล
- Frontend เรียก API ผิดปลายทาง วิธีแก้: สร้าง config.js และกำหนด URL ของแต่ละ service ให้ชัดเจน เช่น AUTH_URL, TASK_URL, USER_URL แล้วแก้ frontend ให้เรียกแต่ละ service โดยตรง
  
## สิ่งที่ได้เรียนรู้จากงานนี้
งานนี้ทำให้ได้เรียนรู้การออกแบบระบบแบบ Database-per-Service มากขึ้น โดยแต่ละ service ต้องมีฐานข้อมูลของตนเองและไม่ควรพึ่งพาการ query ข้ามกันเหมือนระบบเดิม นอกจากนี้ยังได้เข้าใจการใช้ JWT สำหรับ authentication และการนำข้อมูลใน token ไปใช้ควบคุมสิทธิ์การเข้าถึงของผู้ใช้ในแต่ละ service อีกด้วย
